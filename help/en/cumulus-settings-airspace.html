Settings

Airspaces

The table shows you which kind of airspaces can be drawn onto the map. Check the items of the airspaces to be displayed. To change the color of the airspace border or the airspace area touch the related field in the table. That will opens a color dialog where you can change the current color. Pressing button Color Defaults will assign the programmed default colors to all airspaces.

Note that Cumulus will only warn you for airspaces that are drawn.

You will get optical and acoustical warnings if you are near, very near or inside of specified airspaces. If you press the button Airspace Warnings you can set lateral and vertical distances (separately for above and below) that will be applied when generating warnings. If the distance to an airspace is smaller than the specified value, the position is considered near respectively very near to the airspace.

If you press the button Install Airspace a new dialog box is opened. In that box you can define a proxy by pressing button Set Proxy if you need one for Internet access. Beside the label Airspace Url you can enter an url pointing to an airspace file. The url must consist of the following items:

  • Scheme, like http
  • Hostname
  • Pathname
If one of them is missing the url is rejected. Furthermore the file to be downloaded must be a normal file, not compressed, and must carry the extension .txt to be read in after download.

If you press the button Airspace filling you can enable, that the airspaces are filled with a transparent color with different transparency according to the near of the airspace. If you disable the airspace filling, a fixed pattern is used for filling.

The units used for the distances can be configured via the Units page. The corresponding item is Altitude. If unit meters is selected, meter will be used as distance otherwise food.
