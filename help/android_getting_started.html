Getting started with Android

Preconditions

  • The usage of Cumulus on an Android device is only useful with a screen of at least 800 x 480 pixels in landscape mode. Furthermore we recommend a screen diagonal greater than 4" otherwise you have problems to use your fingers for the operation.
  • The device must run Android Release 2.2.x (Froyo) or higher.
  • The screen should be good readable in sunshine.
  • An external storage medium (SD-Card) is needed for storing of map and flight data.
  • If no internal GPS receiver is available Bluetooth radio is a must for connection to external BT GPS devices.
  • Optional your device should have a Wireless Ethernet connection to access the Internet for data downloads.
  • During longer flights you need an external power supply.
  • A device mount is recommended for usage in flight.

Package download and installation

The last release package is to find here. Download the package Cumulus4Android-release.apk and install it on your device. Because the package is not provided on the Google market, you have to enable the installation of non market applications before to get installed Cumulus.

Permissions required by Cumulus

Cumulus requires the following system permissions:
  • Location service updates
  • Internet access
  • Bluetooth and Bluetooth administration
  • Wakelock
  • Read and write access for the SD-card

First startup of Cumulus

On the first run of Cumulus you should have an Internet connection running because Cumulus requires the installation of the Ministro App. Ministro is the Qt library distributor. If you cancel the installation, Cumulus is not runable. Furthermore the needed maps and the Welt2000 data file are automatically dowloaded from the KFLog server, if you allow that.

Configuration of Cumulus

If you set your home position and your flying radius you should have an Internet connection running that all the required maps can be downloaded automatically. The maps and all your other data are installed at the SD-card under the root path /sdcard/Cumulus.

Airspace files have to be installed in /sdcard/Cumulus/maps/airspaces. You can download them by using your Internet browser and as destination that directory.

GPS activation

When Cumulus is ready with its startup, it will be in non-GPS mode. Do all your flight planning work in this mode. To switch on the GPS, press the Android menu button and select the entry GPS on. You can choose then between the internal GPS or a Bluetooth GPS. A BT GPS must be paired before it can be used by Cumulus. Only via Bluetooth radio you can connect external GPS devices.

Usage note

The ported Qt part has some restrictions in its usage. Therefore some GUI elements are not optimal to operate.

Remarks to Android

A big problem is the variety of OS releases and that the manufacturers can modify the Android OS provided by Google. That can cause additional problems during run-time and makes a testing not more possible by us on every device and for every OS release. Also the native Qt library port for Android is not yet error free.

If you step in problems with the GUI surface you can install the app Ministro configuration tool. In that App you can change the used Qt library source. Normally stable is used. Try out testing or unstable and see if the observed issue is fixed.
