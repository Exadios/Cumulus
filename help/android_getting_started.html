Getting started with Android

Introduction

Cumulus is a well established Open Source flight navigation software made for (but not limited to) soaring pilots written in C++. The primary hardware targets were Sharp's Zaurus devices and later Nokia's Internet tablets. Both are able to run Qt software with relatively little adaptation efforts which made it possible to use large parts of the desktop flight preparation software KFlog, created for the KDE framework. The Android Lighthouse project provides a development kit, called Necessitas, for integrating Qt C++ applications into the Android environment with little or no changes, embedded in as native libraries controlled by the JNI API (Java Native Interface).

Preconditions

  • The usage of Cumulus on an Android device is only useful with a screen resolution of at least 800 x 480 pixels in landscape mode. Furthermore we recommend a screen diagonal greater than 4" otherwise you have problems to use your fingers for the operation.
  • The device must run Android Release 2.2.x (Froyo) or higher.
  • The screen should be good readable in sunshine.
  • An external storage medium (SD-Card) is needed for storing of map and flight data.
  • If no internal GPS receiver is available Bluetooth radio is a must for connection to external BT GPS devices.
  • Optional your device should have a Wireless Ethernet connection to access the Internet for data downloads.
  • During longer flights you need an external power supply.
  • A device mount is recommended for usage in flight.

Package download and installation

The last release package is to find on Google play as Cumulus from kflog.org. Download the package and install it on your device.

Permissions required by Cumulus

Cumulus requires the following Android system permissions:
  • Location service updates (GPS)
  • Internet access
  • Bluetooth and Bluetooth administration
  • Wakelock
  • Read and write access for the SD-card

First startup of Cumulus

On the first run of Cumulus after the installation you should have an Internet connection running because Cumulus requires the installation of the Ministro App.

Ministro is a system wide Qt shared libraries installer and provider service. It acts as a bridge between your App and the needed Qt libraries.

If you cancel the installation, Cumulus is not runable. So long Cumulus is installed at your device you should not remove the Ministro App. Otherwise you will be requested to reinstall it.

Furthermore the needed maps and the Welt2000 data file are automatically dowloaded from the Internet, if you allow that.

Configuration of Cumulus

If you set your home position and your flying radius you should have an Internet connection running that all the required maps can be downloaded automatically. The maps and all other data as waypoint files, IGC logger files, flight task file, a.s.o are installed at the SD-card under the root path /sdcard/Cumulus. You can access this directory also from your PC, if you couple your device with it via USB.

Airspace files have to be installed in /sdcard/Cumulus/maps/airspaces. You can download them by using your Internet browser and as destination that directory. On the airspace setting page you can determine which airspace files should be loaded during run-time by Cumulus.

GPS activation

When Cumulus is ready with its startup, it will be in non-GPS mode. Do all your flight planning work in this mode. To switch on the GPS, press the Android menu button and select the entry GPS on. You can choose then between the internal GPS or a Bluetooth GPS. A BT GPS must be paired before it can be used by Cumulus. Only via Bluetooth radio you can connect to an external GPS device.

Usage note

The native Qt GUI part has some restrictions in its usage. Therefore some GUI elements, e.g. color or font dialogs, are not optimal to operate.

Cumulus provides an online help. Please have a look into it, if you have questions about the usage. The Online Help is also to find on the Cumulus web page in two languages, in English and German.

Remarks to Android

A big problem is the variety of OS releases and that the manufacturers can modify the Android OS provided by Google. That can cause additional problems during run-time and makes a testing not more possible by us on every device and for every OS release. Also the native Qt GUI library port for Android is not yet error free.

If you step in problems with the GUI surface you can install the App Ministro configuration tool. In that App you can change the used Qt library source. Normally stable is used. Try out testing or unstable and see if the observed issue is fixed.
